- Reactive state: `loading` (boolean), `errorMessage` (string | null)
- handleLogin() function:
  1. Set loading = true, errorMessage = null
  2. Show "正在获取授权..." toast
  3. Call `uni.login({ provider: 'weixin' })` to get code
  4. Call `wechatLogin(code)` from frontend/src/api/auth.ts
  5. On success: `setToken(token)`, `setUserInfo(user)`, redirect to home
  6. On failure: set errorMessage, show error toast
  7. Finally: set loading = false
- Check for `redirect` query parameter on page load; if present, redirect there after successful login

**Styles:**
- Use CSS variables for colors
- Button custom styling: background #07C160, color white, border-radius 48rpx
- Per UI-SPEC spacing: page padding 64rpx (xl), logo margin-bottom 128rpx (3xl)
- Privacy notice at bottom, muted styling
- Touch target minimum 88rpx for button

Error handling mapping (per UI-SPEC):
- Network error -> "网络连接失败" / "请检查网络设置后重试"
- User cancelled -> "登录已取消" / "请点击'微信登录'按钮重新开始"
- Authorization failed -> "授权失败" / "微信授权验证未通过，请重试"
- Server error -> "服务暂时不可用" / "请稍后重试，或联系技术支持"

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wx.login() code | Temporary code obtained from WeChat client — sent to backend over HTTPS only |
| Token storage | Stored in uni.setStorageSync — mini-program sandbox, not accessible to other apps |
| Privacy policy links | Placeholder links for Phase 2; must point to real policies before production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-15 | Information Disclosure | login.vue token storage | mitigate | Token stored via uni.setStorageSync (sandboxed); no token logged to console; no token in URL params |
| T-02-16 | Spoofing | wx.login code | mitigate | Code is single-use, short-lived (5 min); only valid when exchanged server-side with app secret |
| T-02-17 | Eavesdropping | login.vue -> backend | transfer | HTTPS required in production; BASE_URL must point to HTTPS endpoint (configured via env) |
</threat_model>
