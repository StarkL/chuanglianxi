<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (mini program) -> API | Untrusted input from WeChat mini program |
| Environment variables -> Application | Secrets and configuration loaded from .env |
| File system -> Application | .gitignore must prevent secret leakage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | .env files | mitigate | .gitignore includes .env and .env.local; .env.example documents required vars without real values |
| T-01-02 | Information Disclosure | Error responses | mitigate | Fastify logger enabled but error details not exposed in API responses; production mode suppresses stack traces |
| T-01-03 | Tampering | CORS configuration | mitigate | CORS origin controlled via env var; production default restricts to known WeChat domains, not wildcard |
| T-01-04 | Elevation of Privilege | Database connection | mitigate | DATABASE_URL stored in env, never hardcoded; connection uses least-privilege DB user (not superuser) |
</threat_model>
