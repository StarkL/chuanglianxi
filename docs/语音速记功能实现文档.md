# 语音速记功能实现文档

> 实现日期：2026-06-09  
> 平台：H5 优先（微信小程序需使用同声传译插件）

---

## 📋 功能概述

语音速记功能允许用户通过语音快速记录与联系人的交互，系统自动：
1. **语音转文字** - 使用 Web Speech API
2. **AI 结构化提取** - 使用 qwen3.7-plus 提取关键信息
3. **自动归档** - 保存到交互记录
4. **智能提醒** - 自动创建跟进提醒

---

```
│  │  POST /api/voice-note/save        │  │
│  │  ↓                                 │  │
│  │  保存到 Interaction + Reminder 表   │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

## 🚀 使用指南

### 1. 启动开发环境

```bash
# 启动后端
cd backend
pnpm dev

# 启动前端 (H5)
cd frontend
pnpm dev:h5
```

### 2. 访问语音速记页面

**H5 版本**：
- 打开浏览器访问：`http://localhost:5173/pages/voice-note/voice-note`
- **推荐使用 Chrome 或 Edge 浏览器**（Web Speech API 支持最好）

### 3. 使用流程

#### 步骤 1：录音
1. 点击大圆形录音按钮 🎙️
2. 说出你与联系人的交互，例如：
   > "刚才跟腾讯的李强喝了茶，他说他们团队最近在招高级前端。另外他下个月要去新加坡度假。提醒我 3 周后问问他前端招聘进度。"
3. 点击停止按钮 ⏹

#### 步骤 2：AI 处理
系统自动调用 qwen3.7-plus 提取：
- ✅ 联系人：李强
- ✅ 公司：腾讯
- ✅ 摘要：喝茶交流，团队招聘高级前端
- ✅ 要点：下月去新加坡度假
- ✅ 提醒：3 周后跟进前端招聘进度

#### 步骤 3：选择联系人
- 输入联系人 ID（TODO: 实现联系人选择器）

#### 步骤 4：保存
- 点击"保存语音笔记"按钮
- 自动创建交互记录 + 提醒

## 🔧 API 接口说明

### 1. 处理语音转录文本

**POST** `/api/voice-note/process`

**请求体**：
```json
{
  "transcript": "刚才跟腾讯的李强喝了茶，他说他们团队最近在招高级前端...",
  "contactId": "可选，如果已选择联系人"
}
```

**响应**：
```json
{
  "success": true,
  "data": {
    "extracted": {
      "contactName": "李强",
      "company": "腾讯",
      "title": null,
      "interactionType": "meeting",
      "summary": "喝茶交流，团队招聘高级前端",
      "keyPoints": ["下月去新加坡度假"],
      "reminder": {
        "action": "跟进前端招聘进度",
        "daysLater": 21
      }
    },
    "contact": null,
    "originalTranscript": "刚才跟腾讯的李强喝了茶..."
  }
}
```

### 2. 保存语音笔记

**POST** `/api/voice-note/save`

**请求体**：
```json
{
  "contactId": "uuid",
  "transcript": "原始语音转录文本",
  "summary": "AI 提取的摘要",
  "keyPoints": ["要点1", "要点2"],
  "reminderAction": "跟进前端招聘进度",
  "reminderDays": 21
}
```

**响应**：
```json
{
  "success": true,
  "data": {
    "interaction": {
      "id": "uuid",
      "contactId": "uuid",
      "type": "voice_note",
      "content": "【摘要】...\n【要点】...\n【原文】...",
      "occurredAt": "2026-06-09T10:00:00.000Z"
    },
    "reminder": {
      "id": "uuid",
      "message": "语音笔记提醒：跟进前端招聘进度",
      "scheduledAt": "2026-06-30T10:00:00.000Z"
    }
  }
}
```

## 📱 微信小程序实现方案

微信小程序不支持 Web Speech API，需要使用**微信同声传译插件**：

### 1. 添加插件

在 `frontend/src/manifest.json` 中：
```json
{
  "mp-weixin": {
    "plugins": {
      "WechatSI": {
        "version": "0.3.5",
        "provider": "wx069ba97219f66d99"
      }
    }
  }
}
```

### 2. 修改录音逻辑

```typescript
// 小程序端使用插件
const plugin = requirePlugin('WechatSI')

plugin.voiceTranslator.translate({
  lang: 'zh_CN',
  content: audioFilePath,
  success: (res) => {
    const transcript = res.transTxt
    // 调用后端 /api/voice-note/process
  }
})
```

## 💰 成本分析

### 当前方案（Web Speech API + qwen3.7-plus）

| 项目 | 成本 |
|------|------|
| 语音识别 | **免费**（浏览器原生） |
| AI 提取 | ¥0.008/千 tokens |
| 单次语音速记 | ≈ 1500 tokens ≈ **¥0.012** |
| 1000 次/月 | **¥12/月** |

### 未来方案（qwen3.5-omni-flash）

| 项目 | 成本 |
|------|------|
| 语音+AI 一体化 | ¥0.8/百万 tokens |
| 单次语音速记 | ≈ 2000 tokens ≈ **¥0.0016** |
| 1000 次/月 | **¥1.6/月** |

**结论**：当前方案成本可控，未来升级 omni 模型可进一步降低成本。

## 🐛 已知问题

1. **浏览器兼容性**：Safari 和 Firefox 支持不完整
