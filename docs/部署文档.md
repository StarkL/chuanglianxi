## 三、部署流程（本地构建 → 推送文件）

> VPS 资源紧张，所有构建在开发机（本地 Windows）完成，只推送运行所需文件。

### 3.1 本地构建

```bash
# 1. 安装依赖（如遇权限问题，用 cmd /c "rd /s /q node_modules" 清理后重装）
cd F:\document\pf\IT\projects\chuanglianxi
pnpm install

# 2. 构建后端
cd backend && pnpm build

# 3. 构建前端 H5
cd ../frontend && pnpm build:h5
```

### 3.2 推送文件到 VPS

```bash
# 推送后端构建产物
scp backend/dist/index.js admin@182.92.95.136:/tmp/index.js
scp backend/dist/index.d.ts admin@182.92.95.136:/tmp/index.d.ts
ssh admin@182.92.95.136 "sudo cp /tmp/index.js /tmp/index.d.ts /home/admin/chuanglianxi/backend/dist/"

# 推送前端 H5 文件（构建产物 → Nginx 实际读取目录）
cd frontend && tar -czf /tmp/clx-frontend.tar.gz -C dist/build h5
scp /tmp/clx-frontend.tar.gz admin@182.92.95.136:/tmp/
ssh admin@182.92.95.136 "sudo rm -rf /usr/share/nginx/html/crm/assets /usr/share/nginx/html/crm/index.html && sudo tar -xzf /tmp/clx-frontend.tar.gz -C /usr/share/nginx/html/crm/ && sudo rm -rf /www/crm/frontend/dist/build/h5 && sudo tar -xzf /tmp/clx-frontend.tar.gz -C /www/crm/frontend/dist/build/"

# 推送 Prisma schema（如有变更）
scp backend/prisma/schema.prisma admin@182.92.95.136:/home/admin/chuanglianxi/backend/prisma/
ssh admin@182.92.95.136 "cd /home/admin/chuanglianxi/backend && sudo npx prisma generate && sudo npx prisma db push"
```

### 3.3 重启服务

```bash
# 停止旧服务
ssh admin@182.92.95.136 "kill \$(pgrep -f 'node dist') 2>/dev/null; echo stopped"

# 启动新服务
ssh admin@182.92.95.136 "cd /home/admin/chuanglianxi/backend && nohup sudo node dist/index.js > /home/admin/chuanglianxi/app.log 2>&1 & echo \$!"

# 验证
ssh admin@182.92.95.136 "sleep 2 && curl -sf http://127.0.0.1:5000/api/health"

# 重载 Nginx（前端更新时）
ssh admin@182.92.95.136 "sudo nginx -s reload"
```

## 四、关键安全配置

### 4.1 后端绑定地址 ⚠️ CRITICAL

```typescript
// backend/src/index.ts:22
// ✅ 正确：仅监听 localhost，外部请求必须经过 Nginx
await app.listen({ port: env.PORT, host: '127.0.0.1' })

// ❌ 错误：暴露所有网络接口，可绕过 Nginx 直连后端
// await app.listen({ port: env.PORT, host: '0.0.0.0' })
```

**验证命令**：
```bash
ssh admin@182.92.95.136 "ss -tlnp | grep 5000"
# 应输出：LISTEN 0 511 127.0.0.1:5000 0.0.0.0:*
```

### 4.2 安全中间件

- **Helmet** — 自动设置安全 HTTP 头（CSP、X-Frame-Options 等）
- **CORS** — 限制跨域来源
- **Nginx 安全头** — HSTS、X-Content-Type-Options 等

### 4.3 API 认证

- 所有 `/api/*` 接口（除 `/api/health`、`/api/auth/*`）均需 JWT 认证
- JWT Secret 在 `.env` 中配置，至少 32 字符

## 五、Nginx 架构

```
用户浏览器
    │
    ▼ HTTPS (443)
┌──────────┐
│  Nginx   │ ← 安全头、限流、SSL 终止
└────┬─────┘
     │
     ├── /crm/*  → 静态文件 (/www/crm/frontend/dist/build/h5)
     │
     └── /api/*  → 反向代理 → 127.0.0.1:5000 (Fastify)
```
