# 常联系 - 部署指南

## 架构

```
                    阿里云 ECS
        ┌──────────────────────────────┐
        │  Nginx (80/443)              │
        │  ┌──────────────────────┐    │
        │  │  前端静态文件 (H5)     │    │
        │  └──────────────────────┘    │
        │  ↓ /api/* 反向代理           │
        │  Backend (Node.js :3000)     │
        │  ├── PostgreSQL (:5432)      │
        │  └── Redis (:6379)           │
        └──────────────────────────────┘
```

## 方式一：一键推送部署（推荐）

在**本地**执行：

```bash
# 第一次使用，需要确保服务器有 SSH 密钥登录
bash push-deploy.sh <服务器IP>
# 例如：
bash push-deploy.sh 47.100.100.100
```

这个脚本会：
1. 本地构建前端 H5 版本
2. 通过 rsync 推送代码到服务器
3. 在服务器上执行 `deploy.sh` 完成部署

## 方式二：手动服务器部署

### Step 1: 服务器环境准备

```bash
# 登录服务器
ssh root@<服务器IP>

# 创建项目目录
mkdir -p /opt/changlianxi
cd /opt/changlianxi
```

### Step 2: 获取代码

```bash
# 从 GitHub 拉取
git clone https://github.com/StarkL/chuanglianxi.git .
# 或从本地推送（用 rsync/scp）
```

### Step 3: 配置环境变量

```bash
# 复制模板
cp .env.example .env

# 编辑环境变量
vim .env
```

**必填项：**
- `POSTGRES_PASSWORD` — 数据库密码（改一个强密码）
- `JWT_SECRET` — JWT 密钥（随机字符串）

**选填项（不填对应功能不可用）：**
- `BAIDU_APP_ID/API_KEY/SECRET_KEY` — 百度 OCR 名片识别
- `OPENAI_API_KEY` — AI 丰富联系人信息
- `WECHAT_APP_ID/APP_SECRET` — 微信小程序登录
- `CORS_ORIGIN` — 前端域名（如 `https://clx.example.com`）

### Step 4: 执行部署

```bash
bash deploy.sh
```

## 常用运维命令

```bash
# 查看服务状态
docker compose -f docker-compose.prod.yml ps

# 查看后端日志
docker compose -f docker-compose.prod.yml logs -f backend

# 查看 Nginx 日志
docker compose -f docker-compose.prod.yml logs -f nginx

# 重启服务
docker compose -f docker-compose.prod.yml restart backend

# 更新代码后重新部署
git pull && bash deploy.sh

# 停止所有服务
docker compose -f docker-compose.prod.yml down

# 进入数据库
docker compose -f docker-compose.prod.yml exec postgres psql -U postgres -d changlianxi

# 进入 Redis
docker compose -f docker-compose.prod.yml exec redis redis-cli
```

## HTTPS 配置（可选）

使用 Let's Encrypt 免费证书：

```bash
# 安装 certbot
apt install certbot python3-certbot-nginx -y

# 获取证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

然后在 `nginx.conf` 中启用 HTTPS。

## 端口说明

| 端口 | 服务 | 说明 |
|------|------|------|
| 80 | Nginx | HTTP 访问，反向代理到后端 |
| 443 | Nginx | HTTPS 访问（配置证书后） |
| 5432 | PostgreSQL | 数据库（生产环境建议关闭外网访问） |
| 6379 | Redis | 缓存/队列（生产环境建议关闭外网访问） |
| 3000 | Backend | 后端 API（仅内部访问） |
