# 「常联系」个人 CRM 产品规划文档

> 文档日期：2026-04-16
> 状态：需求确认阶段

---

## 一、产品概述

### 1.1 一句话定位

一个简单、便宜、AI 原生的个人 CRM 微信小程序，专注于**"记住你见过谁、聊了什么、该什么时候联系"**。

### 1.2 产品名称

- **中文名**：常联系
- **英文名**：ChangLianXi

### 1.3 核心价值主张

不做管道、不做交易追踪、不做邮件营销——只做关系维护。让每个人都能轻松维护重要人脉，不让任何一段关系自然流失。

### 1.4 目标用户

- 商务人士：销售、BD、创业者
- 自由职业者/顾问
- 职场新人到中层管理者
- 重视人脉关系的年轻群体（25-40 岁）

## 二、市场分析

### 2.1 市场空白

基于对 Dex、Clay、Monica、folk 等西方产品以及懒牛、悟空 CRM 等中国产品的调研，发现以下空白：

| 空白 | 说明 |
|------|------|
| 中国 AI 原生个人 CRM | 懒牛有 OCR 但无 AI，悟空偏商业 CRM |
| 微信生态深度集成 | 西方产品不支持微信，中国产品 AI 能力弱 |
| 低价位 + 全功能 | $9-20/月对国内用户太贵 |
| 零手动录入体验 | 手动录入是用户流失的最大原因 |
| 语音交互捕获 | 会后语音→结构化数据，中国市场无人做 |

### 2.2 竞品对标

| 维度 | 懒牛 | Dex | Monica | 常联系 |
|------|------|-----|--------|--------|
| AI 能力 | 无 | 基础 | 无 | 深度（分阶段） |
| 微信集成 | 部分 | 无 | 无 | 深度 |
| 定价 | 不透明 | $12/月 | $9/月 | 免费 + ¥15-30/月 |
| 移动体验 | Android 为主 | iOS/Android | Web only | 微信小程序 |
| 名片 OCR | 有 | 有 | 无 | 有 |
| 语音输入 | 无 | 无 | 无 | 有（V2） |
| 关系评分 | 基础统计 | 无 | 无 | AI 评分（V2） |

## 三、产品决策

### 3.1 核心决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 目标市场 | 中国市场优先 | 微信集成是最大差异化，用户获取成本低 |
| 产品形态 | 微信小程序（V1） | 开发成本低、无需下载、天然微信生态 |
| 微信聊天处理 | 手动转发 + AI 提取 | 合规优先，用户完全可控 |
| 商业模式 | 免费 + 增值订阅 | 降低获客门槛，AI 功能付费解锁 |
| 前端框架 | uni-app | 一次开发多端发布，国内生态最成熟 |
| 后端技术 | Node.js + TypeScript | 前后端统一语言，开发效率高 |
| 数据库 | PostgreSQL | JSON 支持好，AI 向量扩展成熟 |
| AI 模型 | 国内大模型 | 合规、成本低、中文能力强 |

### 3.2 AI 深度

**终局目标：深度 AI**，但分三阶段逐步交付。

## 四、版本规划

### 4.1 V1 - MVP（1-2 个月）

**目标：快速上线，验证核心价值**

#### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 微信登录 | 一键授权登录，获取用户基本信息 | P0 |
| 联系人管理 | 增删改查联系人，基础信息录入 | P0 |
| 名片 OCR | 拍照识别名片，自动创建联系人档案 | P0 |
| 交互记录 | 手动记录与联系人的每次交互（时间、方式、内容） | P0 |
| 关系提醒 | 自定义联系频率，到期推送提醒 | P0 |
| 联系人分组 | 标签系统，按工作/朋友/家人等分类 | P1 |
| 通讯录导入 | 一键导入手机通讯录 | P1 |
| 农历生日提醒 | 支持农历生日，自动提醒 | P1 |
| 基础搜索 | 按姓名/公司/标签搜索联系人 | P1 |
| 名片墙 | 查看所有已录入的名片 | P2 |

#### AI 能力（V1 轻量）

| 能力 | 描述 |
|------|------|
| 名片智能识别 | OCR + AI 补全信息（公司官网、职位推断） |
| 基础关系提醒 | 基于最后联系时间的智能提醒（不是简单倒计时） |

#### 免费功能

- 最多 100 个联系人
- 基础联系人管理
- 名片 OCR（每天 10 次）
- 基础关系提醒

#### 付费功能（¥15/月 或 ¥99/年）

- 无限联系人
- 无限名片 OCR
- 高级标签系统
- 农历生日提醒
- 导出功能

### 4.2 V2 - AI 增强（2-3 个月）

**目标：引入 AI 能力，建立差异化壁垒**

#### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 会后语音笔记 | 说句话就能记录，AI 转录 + 结构化 | P0 |
| 关系健康评分 | AI 分析互动频率/内容深度，给每段关系打分 | P0 |
| 智能破冰建议 | 基于历史交互生成个性化开场话题 | P0 |
| 微信聊天记录分析 | 手动转发聊天记录，AI 自动提取联系人和交互摘要 | P0 |
| 联系人自动丰富 | 从公开信息补充联系人背景 | P1 |
| 关系网络图谱 | 可视化展示联系人之间的关联 | P1 |
| 智能联系建议 | AI 推荐"这周该联系谁" | P1 |

#### AI 能力

| 能力 | 描述 |
|------|------|
| 语音转文字 | ASR 将语音笔记转为结构化文本 |
| 聊天摘要 | 从转发的微信聊天中提取关键信息 |
| 关系评分 | 多维度分析关系健康度 |
| 破冰生成 | 基于上下文生成个性化建议 |

#### 付费升级（¥25/月 或 ¥199/年）

- 无限语音笔记
- 关系健康评分
- 微信聊天分析（每月 50 次）
- 智能破冰建议
- 关系网络图谱

---

### 4.3 V3 - 深度 AI（3-4 个月）

**目标：AI 原生体验，不可替代**

#### 功能清单

| 功能 | 描述 | 优先级 |

| 人脉推荐 | 基于现有网络，推荐该认识的人 | P1 |
| AI 周报 | 每周自动生成关系维护报告 | P1 |

#### AI 能力

| 能力 | 描述 |

| 情报生成 | 会前快速生成联系人档案 |

### 5.2 技术栈明细

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | uni-app + Vue 3 + TypeScript | 多端发布，微信优先 |
| UI 组件 | uni-ui 或 uView | 成熟的移动端组件库 |
| 后端 | Node.js + Express/Fastify + TypeScript | RESTful API |
| 数据库 | PostgreSQL 15+ | 主数据存储，JSONB + pgvector |
| 缓存 | Redis | Session、热点数据、限流 |
| 消息队列 | BullMQ | 异步任务（语音转录、AI 分析） |
| 文件存储 | 阿里云 OSS / 腾讯云 COS | 名片图片、语音文件 |
| AI - LLM | 通义千问 / 智谱 GLM | 聊天摘要、关系评分、破冰建议 |
| AI - ASR | 讯飞 / 阿里云 | 语音转文字 |
| AI - OCR | 百度 OCR / 微信 OCR | 名片识别 |
| 支付 | 微信支付 | 小程序内订阅支付 |
| 推送 | 微信服务通知 | 关系提醒推送 |
| 部署 | 腾讯云 / 阿里云 | 国内合规服务器 |

## 八、合规与安全

### 8.1 数据合规

- 用户数据存储在境内服务器（符合《个人信息保护法》）
- 微信聊天记录处理采用手动转发，用户完全可控
- 不提供数据导出至境外的功能
- 明确的数据删除和导出机制

### 8.2 隐私保护

- 最小化数据收集原则
- 用户可随时删除自己的数据
- 聊天记录处理后不保留原始内容，仅保留结构化摘要
- 语音文件转录后可选删除原始音频

## 九、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 微信政策变化 | 小程序被封禁或功能受限 | 保留 uni-app 能力，可快速切到独立 App |
| AI 成本高 | 免费用户亏损 | 严格限制免费用户 AI 调用次数 |
| 用户增长慢 | 现金流断裂 | V1 快速上线，控制开发成本 |
| 大厂跟进 | 竞争加剧 | 先发优势 + 深耕细分场景 |
| 合规风险 | 数据处理违规 | 咨询法律顾问，确保符合 PIPL |
